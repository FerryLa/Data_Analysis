# 릴리스 노트

## 버전 1.0.0 (2026-01-24)

### 🎉 최초 릴리스

2030 해상 통신 시뮬레이션 플랫폼의 첫 번째 정식 릴리스입니다.

---

### ✨ 주요 기능

#### 1. 실시간 AIS 데이터 통합
- AISStream WebSocket 클라이언트 구현
- Q-Max LNG 선박 14척 MMSI 필터링
- 비동기 연결 및 자동 재연결
- 지수 백오프 재연결 전략

#### 2. Dead Reckoning 예측 엔진
- 대권항법 (Great Circle Navigation) 구현
- 시간 기반 오차 누적 모델
- 95% 신뢰 구간 계산
- 바람/해류 드리프트 모델링
- Haversine 거리 계산
- 방위각 계산

#### 3. 선박 시뮬레이션

**암모니아 연료선 (5척)**:
- 경로 기반 웨이포인트 항해
- 5개 주요 항로 사전 정의
  - 중동 → 한국
  - 호주 → 일본
  - 미국 걸프 → 유럽
  - 칠레 → 중국
  - 노르웨이 → UAE
- 확률적 신호 블랙아웃
- 현실적인 속도 프로파일
- 거리 기반 자동 감속

**SMR 추진 선박 (1척)**:
- 통로 제약 기반 항해
- Shapely 기반 지오펜스
- 3가지 구역 타입 (허용/제한/금지)
- 실시간 위반 감지
- 이벤트 로깅

#### 4. 통신 시나리오 시뮬레이션
- Gilbert-Elliott 패킷 손실 모델
- 3가지 통신 프로파일
  - AIS 지상국 (지연 2000ms)
  - VSAT 위성 (지연 500ms)
  - LEO 위성 (지연 30ms)
- 해상 상태 기반 지연 모델
- 블랙아웃 시뮬레이션
- 신호 가용성 지수 (SAI) 계산

#### 5. 웹 인터페이스 (Streamlit)
- Folium 기반 인터랙티브 지도
- 실시간 선박 위치 추적
- Dead Reckoning 오차 반경 시각화
- 통신 성능 대시보드
- 이벤트 로그 테이블
- 시나리오 제어 패널

### 📐 수학적 모델

#### Dead Reckoning 공식

```
위도 예측:
φ₂ = asin(sin(φ₁)·cos(δ) + cos(φ₁)·sin(δ)·cos(θ))

경도 예측:
λ₂ = λ₁ + atan2(sin(θ)·sin(δ)·cos(φ₁), cos(δ) - sin(φ₁)·sin(φ₂))

여기서:
  φ = 위도 (라디안)
  λ = 경도 (라디안)
  δ = 각거리 = d/R (d=거리, R=지구 반경)
  θ = 진침로 (라디안)
```

#### 오차 누적 모델

```
σ_total(t) = √(σ_센서² + σ_침로²·t² + σ_속도²·t²)

구성 요소:
  σ_센서: 기본 AIS 센서 정확도 (~10m)
  σ_침로: 침로 불확실성 (±2°)
  σ_속도: 속도 불확실성 (±0.1 knots)
  t: 마지막 위치 수정 이후 경과 시간
```

#### 신뢰 구간

```
50% 신뢰 반경 (CEP): r₅₀ = 0.67 · σ_total
95% 신뢰 반경: r₉₅ = 2.45 · σ_total
```

- ✅ 통신 지연 영향 측정

### 📝 알려진 제약사항

1. **실시간 AIS 데이터**
   - AISStream 무료 티어 제한 적용
   - Q-Max MMSI 리스트는 예시 (실제 배포 시 교체 필요)

2. **시뮬레이션 데이터**
   - 암모니아 및 SMR 선박은 완전 합성
   - 실제 선박이나 운영을 나타내지 않음

3. **SMR 선박**
   - 가상 시나리오 (현재 상용화 안됨)
   - 연구 목적으로만 설계

4. **Dead Reckoning 정확도**
   - 실제 환경 조건을 완벽히 반영하지 못함
   - 장시간 예측 시 오차 증가

5. **지오펜스**
   - 실제 해상 규제 정책을 대표하지 않음
   - 연구 및 시연 목적
